探索CSS水平垂直居中布局

发表于 2017-03-30 | 分类于 前端 | | 阅读次数

写前端写了近半年，总有一些CSS的小问题等待解决，这几天有空就解决一下这些问题。

本文主要列出CSS实现水平垂直居中的几种方法，以及不同方法的优缺点和适用场景以便读者参考使用。

水平居中

1.元素固定宽度 + margin

优点：实现简单，兼容性好

缺点：宽度固定，扩展性差

适用场景：单个宽度固定元素水平居中

2.inline-block + text-align

优点：实现简单，兼容性好，扩展性好

缺点：当需要居中的元素很多时，设置为inline-block会因为回车符带来一点空白间距

适用场景：单个宽度不固定元素、单个宽度固定元素以及多个元素的水平居中

3.浮动 + 绝对/相对定位

优点：兼容性好，扩展性好

缺点：实现复杂，不好理解，child1元素不能设任何明显样式，parent如果没设overflow: hidden，当视窗过窄时会出现滚动条

适用场景：单个宽度不固定元素、单个宽度固定元素以及多个元素的水平居中

4.绝对/相对定位 + 负的margin值

优点：实现简单，兼容性好

缺点：不好理解，宽度固定（margin的值必须为元素宽度的一半），扩展性差

适用场景：单个宽度固定元素的水平居中

5.绝对定位 + transform

优点：实现简单，扩展性好

缺点：低版本浏览器不支持CSS3属性，不太好理解

适用场景： 单个宽度不固定元素或单个宽度不固定元素的水平居中

6.fit-content + margin

优点：实现简单，扩展性好

缺点：存在兼容性问题

适用场景：单个宽度不固定元素或单个宽度固定元素的水平居中

7.flex

优点：实现简单，扩展性好

缺点：存在兼容性问题

适用场景：单个宽度不固定元素或单个宽度不固定元素的水平居中

垂直居中

1.table-cell + vertical-align

优点：实现简单，扩展性好

缺点：IE8以下不支持（可替换为table布局）

适用场景：单个高度不固定元素、单个高度固定元素以及多个元素的垂直居中

2.绝对定位 + 负的margin值

c优点：兼容性好

缺点：高度固定，扩展性差

适用场景：单个高度固定元素的垂直居中

3.绝对定位 + transform

优点：实现简单

缺点：低版本浏览器不支持CSS3属性，不太好理解

适用场景：单个高度不固定元素、单个高度固定元素的垂直居中

4.在居中元素之前加一个浮动元素

优点：兼容性好

缺点：不好理解，实现复杂，高度固定，扩展性差

适用场景：单个高度固定元素的垂直居中

5.相对定位 + calc()数值计算

优点：实现简单

缺点：高度固定，扩展性差

适用场景：单个高度固定元素的垂直居中

6.绝对定位 + margin + top, bottom

优点：实现简单

缺点：IE8以下不支持，高度固定，扩展性差

适用场景：单个高度固定元素的垂直居中

7.伪元素 + inline-block + vertical-align

优点：我想不到有啥优点T T，这种方法建议别用

缺点：不好理解，实现复杂，当文本超过一行（超出父元素宽度需要换行时）就没办法居中了，而且伪元素在IE8以下不支持

适用场景：单个元素水平居中

8.flex

优点：同水平居中方法7

缺点：同水平居中方法7

适用场景： 同水平居中方法7

总结：

以上是一些奇奇怪怪的CSS实现水平垂直居中的方法，感觉以后会越来越多去用flex布局。

因为其中一些奇怪的方法虽然能实现居中，但是扩展性不强还不好理解，并且现在IE8以下的兼容开始不被纳入业务中了，所以简单有效的flex布局应该会越来越普及。
