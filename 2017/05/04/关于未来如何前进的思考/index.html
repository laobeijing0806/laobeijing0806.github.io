关于未来如何前进的思考

发表于 2017-05-04 | 分类于 生活 | | 阅读次数

今后我会一直做前端写代码吗？

这个问题来自今天看到的一篇公众号推文技术人的未来在哪里？，里面提到了我从开始决定当一名程序员就在思考问题：我会不会写一辈子代码？显然，答案是不会。

如何规划将来的职业发展

在知乎上也经常看过类似回答：许多人写了多年代码之后，会转型做管理岗，比如当leader去带别人做项目，自己不做底层敲代码的事情，而是更多地去思考项目架构上的问题。这种是比较常见和容易做到的，而有一些做得比较好的大牛级人物则会继续深入某个领域，去钻研和深入理解某种技术，这些人往往是行业里的代表人物。

现在似乎是两条路摆在我面前，一是多年后转管理岗；二是做一名深入研究某个领域的专家。这两者无论哪条路都需要自己先在前端这个领域不断地学习和积累，需要不断地试错和锻炼，等技术达到了相当的高度，才是做选择的时候。但是我这个人有个不好的习惯——做事情往往喜欢规划好了再按照计划开始走下去。为什么说这个习惯不好，因为我常常因此想得太多而做得太少，很多时候只顾着思考后面会发生什么而忽略了实际操作的可能性，这导致自己经常提出许多不切实际的幻想，最终又因为难以实现而放弃，所以现在我依旧时不时会陷入这种无意义的幻想中。但是自从写前端之后，我偶尔会发现自己之前的一些想法其实在不断实践中发现其实是可行的。因此我渐渐开始学会放弃部分无意义的幻想，转而去实现有意义且有可能实现的想法。

决定给自己更多可能

首先，从小到大我对许多新奇的事物都充满了好奇，对于不理解的新鲜事物我都保持着开放和包容的心态，因为我觉得任何领域都有值得学习的东西。也正因为如此，我对于某一特定领域难以保持长期的专注，所以去钻研和深入理解并不是我的兴趣所在。

其次，上面的那篇推文里写到了：

多领域、多维度发展的人才才是技术人的未来，也才是未来人才的发展方向。

这句话我突然想到了冯巩曾经在一个相声里有句台词：

我是说相声里歌唱的最好的，唱歌里相声说的最好的。

虽然这讽刺的是某些人对于知识的学习杂而不精，但是这却是公司梦寐以求的人才，当然，这建立在你的业务水平和其他人差不多的同时你还有其他方面才能的情况下。

因此，无论我今后要去做什么岗位，首先要做好的就是当前的领域，也就是前端。在做好了自己的基本工作之外，我不会给自己设任何限制，我会去学习后端，产品，设计这些方面的知识，这样在平时工作中方便沟(dui)通(ren)。我会尝试去把自己培养成为一个多维度的人才，走出学习的舒适区，去尝试使用不同的技术，获得不同方面的成长，同时也给自己的未来更多可能。

写到这里，关于未来的思路开始变得清晰了：不去永远做底层的码农，努力精进自己前端的技能，同时尽可能学习其他方面的知识，等到技术达到一定高度了转去做管理岗位。这便是我关于未来的部分规划，尽管还有很多东西等待实践去解决，但是目前对于未来我已经变得不那么迷茫了。

目前写了几篇博客，虽然没人看T T，但是这个习惯我会尝试去坚持。因为把自己零碎的想法拼凑成一篇文章并写下来，对自己来说是一种锻炼。这种锻炼不仅是在语言组织方面，还在一定程度上帮助自己克服了爱幻想的坏习惯。总之，写博客给我的帮助颇多，也希望今后自己的文笔和技术能越来越好。

时夜，距离毕业季一年，于家园工作室新机房记录。
